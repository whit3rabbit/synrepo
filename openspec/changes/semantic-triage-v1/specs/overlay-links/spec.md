## MODIFIED Requirements

### Requirement: Define evidence-verified cross-link candidates
synrepo SHALL define a cross-link candidate as a proposed relationship between two graph nodes that is generated by the synthesis pipeline via a two-stage or three-stage triage (cheap deterministic prefilter, optional semantic similarity prefilter when enabled, then LLM-backed evidence extraction), verified against cited evidence through fuzzy LCS matching, scored for confidence, and stored in the overlay as non-authoritative until a human declaration promotes it to the graph. Cross-links never enter the graph directly from the pipeline. The deterministic prefilter SHALL use deterministic signals only (name/identifier match, graph distance cutoff) so candidate generation cost is bounded by repository size, not by LLM throughput. When semantic triage is enabled, the semantic prefilter SHALL run on pairs discarded by the deterministic prefilter and SHALL forward pairs exceeding the configured similarity threshold to the LLM verification stage.

#### Scenario: Generate a candidate cross-link
- **WHEN** the synthesis pipeline infers a possible relationship between a code node and a prose node with cited evidence
- **THEN** the overlay stores the candidate with: source nodes, inferred relationship kind, confidence score and tier, cited evidence spans with LCS ratios, both endpoints' content hashes at generation time, triage source (`deterministic` or `semantic_triage`), and full provenance
- **AND** the stored candidate is labeled non-authoritative and does not modify the graph

#### Scenario: Reject a candidate with no verifiable evidence
- **WHEN** the synthesis pipeline proposes a cross-link but cannot produce cited evidence that verifies above the minimum LCS threshold
- **THEN** the overlay contract requires the candidate to be discarded or held at `below_threshold`
- **AND** the response does not surface an unverified candidate as probable

#### Scenario: Prefilter discards infeasible candidate pairs
- **WHEN** the synthesis pipeline considers a (prose, code) pair whose name-match score is zero and graph distance exceeds the configured cutoff
- **THEN** the deterministic prefilter discards the pair without invoking the LLM
- **AND** the pair is eligible for the semantic prefilter if semantic triage is enabled

#### Scenario: Semantic prefilter rescues a deterministically discarded pair
- **WHEN** semantic triage is enabled and a (prose, code) pair was discarded by the deterministic prefilter
- **THEN** the semantic prefilter computes cosine similarity between the pair's vector representations
- **AND** if similarity exceeds the threshold, the pair is forwarded to LLM evidence extraction with source `semantic_triage`
- **AND** if similarity is below the threshold, no candidate row is written
